Baft checks architecture contracts declared in `BAFT.md` files.

A `BAFT.md` file contains one Mermaid flowchart that:
- defines nodes with globs
- defines allowed imports with arrows
- can mark nodes `:::endophobic` to forbid same-node imports

Files matching `.baftignore` (gitignore syntax) are invisible to Baft entirely.

Typical workflow:
1. Generate a baseline with `baft dump .`
2. Edit `BAFT.md` to express the architecture you want
3. Run `baft check .`
4. Fix violations by adding edges or moving code

Supported languages: Go, TypeScript, Dart, Kotlin, Rust.

Full local manual: `baft manual`
